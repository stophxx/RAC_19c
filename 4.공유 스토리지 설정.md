**※위에 oracleasm init 할때 에러가 난다면**
```
1. 문제 확인

# 현재 커널 확인
uname -r
# 결과: 5.15.0-200.131.27.el8uek.x86_64 (UEK 커널)

# kmoe-oracleasm 패키지를 설치합니다.
yum install kmod-oracleasm

# ASMlib 패키지 확인합니다.(3개가 결과로 나와야합니다)
rpm -qa | grep oracleasm

# ASMlib 초기화 시도 (실패)
oracleasm init
# 에러: Loading module "oracleasm": failed

2. 사용 가능한 커널 확인

 설치된 커널 목록 확인
grubby --info=ALL | grep -E "^kernel=|^title="

3. 기본 커널을 RHCK(4.18.0)로 변경

# 방법 1: 커널 경로로 직접 설정
grubby --set-default=/boot/vmlinuz-4.18.0-513.5.1.el8_9.x86_64

# 방법 2: 인덱스로 설정 (더 확실)
grubby --set-default-index=1

# 설정 확인
grubby --default-kernel
grubby --default-index

4. GRUB 설정 재생성

# GRUB 기본값 확인
cat /etc/default/grub | grep GRUB_DEFAULT

# BIOS 부팅용 GRUB 재생성
grub2-mkconfig -o /boot/grub2/grub.cfg

# UEFI 부팅용 GRUB 재생성
grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg

5. 재부팅
reboot

6. 재부팅 후 확인 및 ASMlib 초기화

# 커널 확인 (4.18.0으로 변경됨)
uname -r

# ASMlib 초기화 (성공!)
oracleasm init

# ASMlib 설정
oracleasm configure -i
 # oracle, dba, y, y 입력

# 상태 확인
oracleasm status

# 구성확인
oracleasm configure
```

- **공유 스토리지 포맷**

VM에 연결된 디스크를 파티션 분할 작업을 수행한다.

디스크 전체(/dev/sdb) 말고 파티션(/dev/sdb1) 단위로 ASM 디스크를 만들겠다.

```
ls /dev/sd*
fdisk /dev/sdb 
fdisk /dev/sdc 
fdisk /dev/sdd 
fdisk /dev/sde 
fdisk /dev/sdf 
ls /dev/sd*
```

명령어 수행 후 순서대로 n, p, 1, 엔터, 엔터, w를 입력한다.(명령어를 한 줄씩 실행한다.)

- **ASM disk 생성**

VM에 추가된 디스크를 각각 ASM이 쓸 디스크로 할당(등록)한다.

```
# root 유저로 수행
oracleasm createdisk CRS1 /dev/sdb1    # /dev/sdb1에 CRS1이라는 ASM 라벨 부여
oracleasm createdisk CRS2 /dev/sdc1
oracleasm createdisk CRS3 /dev/sdd1
oracleasm createdisk DATA /dev/sde1
oracleasm createdisk FRA /dev/sdf1

oracleasm scandisks
oracleasm listdisks
```

<img width="323" height="152" alt="image" src="https://github.com/user-attachments/assets/cf92a849-2f75-4198-a502-6941f44a9c77" />

CRS1, CRS2, CRS3 : Clusterware 관련 파일용(OCR, Voting Disk)

DATA : 데이터파일용

FRA : 아카이브 로그, 백업, 복구 영역용

**정리**

VM에 추가한 공유 디스크를 ASM에서 사용하기 위해 각 디스크에 1개 파티션을 생성한 후,
ASMLib을 통해 CRS1, CRS2, CRS3, DATA, FRA 디스크로 등록하였다.


- **위의 문제가 난 이유는?**
- **근본 원인 분석**
    - **결정적인 문제:** Oracle Linux 8.4와 8.9의 기본 커널이 다릅니다.
    
    | 버전 | 기본 커널 | ASMlib 모듈 |
    | --- | --- | --- |
    | **OL 8.4** | RHCK (4.18.0) 또는 UEK 선택 가능 | `kmod-redhat-oracleasm` 작동 ✅ |
    | **OL 8.9** | **UEK (5.15.0) 기본** | `kmod-redhat-oracleasm` 작동 ❌ |
- **무엇을 잘못했나?**
    1. **버전 차이를 간과함**
        - 8.4 문서를 8.9에 그대로 적용
        - 커널 차이를 확인하지 않음
    2. **잘못된 ASMlib 모듈 설치**
        
        ```bash
        # 설치한 것 (RHCK용)
        kmod-redhat-oracleasm
        
        # UEK를 사용한다면 필요한 것
        kmod-oracleasm  # UEK용
        
        ```
        
    3. **커널 확인을 안 함**
        
        ```bash
        # 설치 전 반드시 확인했어야 할 것
        uname -r  # 어떤 커널인지 확인
        
        ```
        
- **교훈 (OS 버전이 다르면):**
    1. 현재 커널 버전 확인 (`uname -r`)
    2. 해당 버전의 공식 문서 확인
    3. 커널에 맞는 패키지 설치
    4. 또는 커널 통일
- **결론:** "8.4 문서를 맹신"한 게 아니라, "커널 차이를 확인하지 않은 것"이 문제였습니다.
